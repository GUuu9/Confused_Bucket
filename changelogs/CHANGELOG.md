## 💡 기록 규칙
1. 새로운 기능 추가나 변경 사항에 Commit을 수행하면 `changelogs/commit_{번호}.md` 파일을 생성합니다.
2. 각 일별 로그에는 **🚀 주요 업데이트**(기능별 상세 설명)와 **🛠 수정된 파일**(파일 경로 목록) 섹션을 반드시 포함합니다.
3. `CHANGELOG.md` 메인 인덱스 파일에 해당 Commit 요약본과 링크를 추가합니다.
4. commit을 수행할 경우, `feat:[현재 버전 정보] ...` 형태로 작성합니다.
